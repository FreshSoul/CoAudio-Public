# CoAudio-Public
# CoAudio 新手引导

这份引导面向第一次使用 CoAudio 的用户。你可以把它当成从下载安装到完成第一条声音检索、试听、整理、导出和云端协作的完整路线图。

## 0. 先理解 CoAudio 的几个概念

### 本地库

本地库管理你电脑或 NAS 上已有的音频文件。CoAudio 扫描文件夹后只建立索引，不会移动源文件。

适合：

- 个人音效库
- 离线工作
- NAS 或移动硬盘素材
- 快速试听和拖进 DAW

### 音频包

CoAudio 会根据文件所在目录生成音频包视图。比如：

```text
D:\SoundLibrary\Whooshes\Long\
D:\SoundLibrary\Impacts\Metal\
```

这些目录会在侧栏中变成可浏览、可筛选、可分组的包。

### 语义检索

传统搜索依赖文件名和文字字段；语义检索会给音频和文字生成向量。启用后，你可以用自然语言找声音，例如“冰冷的科幻空气释放声”。

语义检索需要先生成 CLAP 嵌入。没生成前，普通搜索仍然可用。

### 元数据

元数据是声音的说明书，包括文件名、描述、关键词、UCS 分类、设计师、项目、厂牌、时长、采样率、声道、BWF/iXML 信息等。

素材越多，元数据越重要。

### 云端库

云端库用于团队共享或跨设备使用。云端分为组织库和个人库：

- 组织库：B 端/团队使用，通常上传源文件到云端。
- 个人库：面向个人分布式使用，可以同步元数据和本机持有关系。

### 合成台

合成台是一个轻量多轨声音设计工作区。你可以输入一句创作描述，让 AI 拆成多个声音层，从素材库里找候选，加入时间线后混合导出。

## 1. 安装和启动

### 普通用户

如果你拿到的是安装包或便携版：

1. 打开 `CoAudio-Setup-版本号.exe` 或 `CoAudio-Portable-版本号.exe`。
2. 安装或直接运行。
3. 启动后，CoAudio 会打开桌面窗口，并自动启动本地音频服务。


## 2. 第一次打开后做什么

第一次打开 CoAudio 时，建议按这个顺序：

1. 添加本地音频文件夹。
2. 等待扫描完成。
3. 先用普通搜索确认素材能搜到。
4. 生成语义索引。
5. 用自然语言搜索。
6. 试听、收藏、编辑元数据。
7. 根据需要配置 AI 和云端。

## 3. 添加本地音效库

1. 在左侧侧栏找到本地库区域。
2. 点击添加文件夹。
3. 选择你的音效库根目录，例如：

```text
D:\SoundLibrary
E:\Projects\GameAudio\SFX
\\NAS\Audio\SFX
```

4. CoAudio 会开始扫描。
5. 扫描完成后，素材会出现在主列表中。

### 扫描时会发生什么

CoAudio 会读取：

- 文件路径
- 文件名
- 扩展名
- 时长
- 采样率
- 声道数
- 格式
- 位深
- 文件大小
- 修改时间
- BWF/iXML/RIFF INFO 等嵌入元数据
- UCS 命名中可解析出的分类信息

它不会移动你的源文件。

### 如果扫描后没有素材

检查：

- 文件夹里是否真的有音频文件
- 文件格式是否被支持
- 外部硬盘或 NAS 是否在线
- 路径权限是否正常
- 设置页里的扫描状态是否报错

## 4. 浏览和筛选素材

主界面通常由几部分组成：

- 顶部：搜索框、AI/语义开关、合成台入口、主题和设置入口
- 左侧：全部资源、收藏夹、最近使用、音频包、分组、云端库
- 中间：结果表格或资源网格
- 右侧：检视器和元数据面板

### 全部资源

显示当前本地库或当前云端库里的所有素材。

### 收藏夹

显示你标记为收藏的素材。适合保存常用爆点、UI、脚步、whoosh、转场等。

### 最近使用

显示最近试听或使用过的素材，方便回到刚刚听过的声音。

### 音频包

按目录组织素材。你可以点击某个包，只看该包下的素材。

### 虚拟分组

你可以给音频包建立分组，例如：

- 常用库
- 项目 A
- 武器
- 环境
- 人群
- UI

虚拟分组不会移动磁盘上的真实文件，只改变 CoAudio 里的组织方式。

## 5. 普通搜索

在搜索框输入关键词，例如：

```text
metal hit
footstep
rain
magic
爆炸
机械
门
```

普通搜索会匹配文件名、描述、关键词、UCS 分类、项目、设计师等字段。

适合：

- 已经知道素材命名规则
- 搜索明确词汇
- 找某个包或某个项目里的文件
- 快速筛掉不相关内容

## 6. 语义检索

语义检索适合“不知道文件怎么命名，但知道想听什么”的场景。

### 先生成语义索引

1. 打开设置页。
2. 找到语义检索/CLAP 相关区域。
3. 点击生成或继续生成嵌入。
4. 等待任务完成。

如果素材很多，第一次生成会比较久。之后新增素材只需要增量处理。

### 语义搜索示例

你可以输入：

```text
潮湿地下室里的远处水滴声
厚重的金属撞击，带很短的尾音
未来科技门打开，干净、低频、机械
巨兽脚步落在泥地上
魔法能量聚集然后爆开
```

打开 AI/语义开关后，搜索结果会按语义相似度返回。

### 查找相似声音

当你听到一条接近目标的声音时，可以使用“查找相似”。CoAudio 会用这条声音的音频向量去找相似素材。

适合：

- 找同一套 Foley 的其他变体
- 找同风格 whoosh
- 找同类 UI 点击
- 找同质感爆炸层
- 给合成台扩展候选素材

## 7. 试听素材

选择一条素材后，右侧检视器会显示波形和播放控制。

常用操作：

- 点击播放/暂停
- 拖动波形定位
- 选择片段区域试听
- 用键盘上下切换素材
- 使用 Shift + 滚轮快速浏览
- 查看多声道波形

### 实时效果

检视器内置：

- EQ
- Pitch
- Volume
- Reverb
- Delay

这些效果用于快速判断素材是否适合，不会自动改写源文件。

例如，你可以：

- 把 whoosh 降几个半音听是否更厚
- 给短撞击加一点混响判断空间感
- 用 EQ 削掉低频判断是否适合 UI
- 快速提高音量听细节

## 8. 拖进 DAW/WWISE

### 本地素材

直接从列表或检视器拖拽到 DAW。CoAudio 会把真实本地文件路径交给 DAW。

### 云端素材

云端素材没有本机固定路径。拖拽时流程是：

1. CoAudio 请求云端下载接口。
2. 云端验证权限。
3. 当前电脑下载素材到临时目录。
4. Electron 把临时文件路径交给 DAW。

如果你没有下载权限，拖拽会失败。

## 9. 编辑元数据

选中素材后，在右侧元数据面板中可以查看和编辑信息。

常见字段包括：

- 描述
- 关键词
- 备注
- 设计师
- 项目/节目
- Library
- UCS Category
- UCS Subcategory
- CatID
- 收藏状态

### 单条编辑

1. 选中素材。
2. 打开右侧元数据面板。
3. 点击编辑。
4. 修改字段。
5. 保存。

### 批量编辑

1. 在结果列表中多选素材。
2. 打开元数据批量编辑。
3. 输入要统一写入的字段。
4. 确认保存。

批量编辑适合：

- 给一批素材加项目名
- 给一个包补关键词
- 统一设计师字段
- 批量设置 UCS 分类

### 云端编辑历史

云端库支持字段级历史。多人协作时，可以查看谁在什么时候把哪个字段从什么值改成了什么值。

## 10. 收藏和最近使用

### 收藏

当你发现常用素材时，可以点击收藏。收藏不会改变文件，只在 CoAudio 里记录。

适合收藏：

- 常用 whoosh
- 常用 UI
- 高频脚步
- 项目基准素材
- 好用的冲击层

### 最近使用

CoAudio 会记录最近使用过的素材，方便你回到刚刚听过或拖过的声音。

## 11. 音频包和虚拟分组

### 音频包

音频包通常来自目录。比如一个文件夹就是一个包。

你可以：

- 点击包筛选素材
- 多选包筛选
- 查看包内数量
- 删除包对应文件
- 将包加入虚拟分组

### 虚拟分组

虚拟分组是 CoAudio 里的整理结构，不改变真实目录。

推荐分组方式：

- 按项目：Project A、Project B
- 按用途：UI、Foley、Ambience、Weapons
- 按常用程度：Favorites、Daily、Archive
- 按质量：精选、待整理、低优先级

## 12. 保存搜索

如果某个搜索经常用，可以保存下来。

例如：

- “雨 夜 城市”
- “metal impact”
- “UI click bright”
- “magic whoosh”
- “需要补元数据”

保存搜索能让你快速回到某个工作上下文。

## 13. 导出和转码

CoAudio 支持批量导出素材。

常见用途：

- 把选中的素材复制到项目目录
- 转成统一格式
- 统一采样率
- 输出给剪辑、动效、外包或引擎团队

可能的导出设置包括：

- 目标目录
- 格式
- 采样率
- 声道数
- 位深

MP3、FLAC、AAC 等格式通常需要 ffmpeg。

## 14. AI 设置

CoAudio 的大模型能力是可选的。即使不配置大模型，也可以使用本地扫描、普通搜索、语义检索、试听、元数据编辑和导出。

配置大模型后，可以使用：

- 自然语言搜索解析
- 批量 AI 打标
- 合成台自动拆层

### 配置步骤

1. 打开设置页。
2. 找到 AI/大模型设置。
3. 选择 provider。
4. 填入 API Key。
5. 填入模型名和可选 base URL。
6. 保存并测试。

API Key 保存在本机配置中。

## 15. AI 批量打标

AI 批量打标可以帮助你为素材生成：

- 描述
- 关键词
- UCS 分类建议

推荐工作流：

1. 先扫描和探测本地库。
2. 生成语义索引。
3. 选中一批需要整理的素材，或选择只处理未打标素材。
4. 启动 AI 批量打标。
5. 抽查结果。
6. 必要时手动修正。

不要把 AI 结果当成永远正确的标准。它适合做第一轮整理和建议，人仍然应该抽查关键资产。

## 16. 合成台入门

合成台是 CoAudio 的轻量声音设计工作区。

### 打开合成台

在顶部搜索框旁点击“合成台”。

### 用一句话开始

输入 brief，例如：

```text
黑暗魔法箭命中石墙，带碎石和低频冲击
未来飞船舱门打开，液压、空气释放、电子提示音
巨型机器人脚步落在湿润混凝土上
```

点击分析后，如果已配置大模型，CoAudio 会把 brief 拆成多个层。

### 查看候选

每个层会自动检索候选素材。你可以：

- 点击候选试听
- 手动修改检索词
- 加载更多候选
- 把候选加入时间线

### 编辑时间线

在时间线里可以：

- 移动片段
- 裁剪左右边缘
- 调整音量
- 调整淡入淡出
- 右键设置循环
- 右键变调
- Mute/Solo 某条轨道
- 缩放时间线
- 调整轨道高度

### 导出组合

1. 点击导出设置。
2. 选择 WAV/FLAC/MP3。
3. 选择声道数、采样率、位深。
4. 点击导出组合。
5. 导出的声音会写入本地库的渲染目录，并重新入库。

建议先用 WAV 24bit/48k 输出，再按项目需要转码。

## 17. 云端账号和团队协作

如果你只用本地库，可以跳过这一部分。

云端协作是 CoAudio 区别于普通本地音效库的关键能力。它不只是把文件上传到服务器，而是把素材、元数据、权限、编辑历史、审计记录和跨设备取流放在同一套协作模型里。

你可以用三个关系理解它：

- C-C：个人到个人。适合创作者之间共享个人库、多设备访问、个人分布式素材流转。
- B-B：组织到组织或团队到团队。适合工作室、项目组、外包团队、声音供应商之间按项目协作。
- B-C：组织到个人。适合组织把项目素材、安全授权素材或精选资产开放给个人创作者、自由职业者、客户审核人员。

### 登录或注册

1. 打开账户/团队面板。
2. 填写云端服务地址，例如：

```text
http://127.0.0.1:8001
```

3. 选择注册或登录。
4. 注册/重置密码时，如果服务器配置了邮箱，会收到验证码。
5. 如果是开发服务器且未配置 SMTP，界面可能回显 dev code。

### 云端库类型

云端库分为：

- 个人库：适合个人分布式库和共享。
- 组织库：适合团队集中管理，通常上传源文件。

### 创建组织

1. 进入账户/团队面板。
2. 打开组织页。
3. 输入组织名称。
4. 创建组织。
5. 创建者默认是 admin。

### 邀请成员

1. 选择组织。
2. 输入成员邮箱。
3. 选择角色：admin、editor、viewer。
4. 发送邀请。
5. 对方输入邀请码接受。

角色大致含义：

- admin：管理组织、库和成员。
- editor：上传和编辑素材。
- viewer：查看、检索、试听，部分下载能力按服务端权限控制。

### 创建组织库

1. 进入组织页或库页。
2. 选择组织。
3. 创建组织库。
4. 将音频上传到组织库。

组织库适合团队共享素材。上传后，授权成员可以从自己的电脑检索、试听和拖拽。

### 上传到云端

有两种常见方式：

**上传入库为组织库素材**  
用于 B 端组织库。源文件会上传到云端存储，成员可以跨设备访问。

**同步到个人云库**  
用于个人分布式库。通常同步元数据和本机持有关系，字节可留在本机。

使用时要注意当前选择的是个人库还是组织库。

## 18. 三种云协作场景：

### C-C：个人到个人

C-C 是最轻量的协作关系。它适合两个或多个个人创作者之间共享声音素材。

典型场景：

- 你在工作站上有一套个人 Foley 库，想让笔记本也能检索和试听。
- 你和另一位声音设计师一起做短片，希望互相开放精选素材。
- 你临时把某个个人库共享给协作者，让对方只在项目周期内使用。
- 你希望字节尽量留在本机，但云端能同步元数据和持有关系。

推荐流程：

1. 登录云端。
2. 创建或使用自己的个人库。
3. 在侧栏选择“同步到个人云库”。
4. 将本地素材的元数据、向量和持有关系登记到云端。
5. 在个人库中生成授权码或共享邀请。
6. 对方登录后接受共享。
7. 对方即可在权限允许范围内检索、试听或取流。

注意：个人分布式库依赖持有节点在线。如果源文件所在电脑离线，对方可能能看到元数据，但无法取到真实音频字节。

### B-B：组织到组织、团队到团队

B-B 是更正式的项目协作关系。它适合有组织边界、角色权限和交付责任的团队工作。

典型场景：

- 游戏公司和外包声音团队共享某个项目库。
- 声音工作室给剪辑、动效、混音团队提供统一素材入口。
- 多个内部项目组各自维护组织库，但希望资产结构统一。
- 供应商交付素材后，甲方需要审计下载、编辑和上传记录。

推荐流程：

1. 创建组织。
2. 创建组织库。
3. 邀请成员加入组织。
4. 给成员分配角色：admin、editor、viewer。
5. editor 上传素材到组织库。
6. 成员按权限检索、试听、下载和编辑。
7. 管理员查看编辑历史和审计记录。

B-B 的重点是“库即边界”。每个组织库都可以代表一个项目、一个部门、一个客户或一个交付阶段。团队不再依赖网盘链接和口头规则，而是通过角色、历史和审计来管理声音资产。

### B-C：组织到个人

B-C 是组织资产进入个人工作台的关系。很多真实项目都需要个人参与：自由声音设计师、远程剪辑师、临时外包、导演、客户审核人员等。

典型场景：

- 工作室把项目音效库授权给自由声音设计师。
- 组织给个人成员开放精选素材库。
- 客户只需要试听或下载某批声音，不应该获得编辑权限。
- 企业素材库希望被个人创作者在桌面端像本地库一样使用。

推荐流程：

1. 组织管理员创建组织库。
2. 上传或整理需要开放的素材。
3. 邀请个人用户加入组织，或通过库共享授予访问。
4. 根据角色控制能力：viewer 试听/查看，editor 可上传/编辑，admin 可管理。
5. 个人用户在桌面端选择该云库。
6. 像使用本地库一样搜索、试听、拖拽，但操作仍受组织权限约束。

B-C 的价值是两边都舒服：个人不需要学习复杂后台，仍然在 CoAudio 桌面端工作；组织不用把整包素材裸发出去，仍然保留权限和审计。

## 19. 个人分布式库和节点

个人分布式库的目标是：素材不一定全部上传到中心，但云端知道谁持有这份素材，以及如何在授权情况下取流。

### 加入共享网络

1. 打开账户/团队面板。
2. 找到节点或 mesh 相关区域。
3. 点击加入共享网络。
4. 本机会向云端注册为一个节点。
5. 同步个人库后，本机会登记自己持有哪些云端资产。

当持有节点在线时，授权用户可以通过云端定位并取流。

### 什么时候会取不到

可能原因：

- 持有素材的电脑离线
- 节点未注册
- 节点心跳停止
- 网络隧道未连接
- 本机文件被移动或删除
- 权限不足

## 20. 审计和历史

云端协作场景下，CoAudio 记录关键操作：

- 上传
- 下载
- 编辑
- 授权
- 接受邀请
- 撤销授权
- 注册
- 重置密码

素材字段编辑会记录版本历史。团队可以知道某个描述、关键词或分类是谁改的。

## 23. 推荐的新手练习

### 练习 1：建一个本地库

1. 添加一个小音效文件夹。
2. 等待扫描完成。
3. 搜索 `hit` 或 `rain`。
4. 试听 5 条素材。
5. 收藏 2 条素材。

目标：熟悉扫描、搜索、试听和收藏。

### 练习 2：整理一个包

1. 在侧栏找到一个音频包。
2. 给它加入虚拟分组。
3. 选中几条素材。
4. 批量写入项目名或关键词。

目标：理解音频包和虚拟分组。

### 练习 3：启用语义检索

1. 生成 CLAP 嵌入。
2. 输入一句自然语言。
3. 找到一条接近的声音。
4. 使用查找相似。

目标：理解语义检索和以音搜音。

### 练习 4：用合成台做一个新声音

1. 打开合成台。
2. 输入一句 brief。
3. 试听候选。
4. 加入 3 条声音到时间线。
5. 调整淡入淡出和音量。
6. 导出 WAV。
7. 在主库中搜索导出的新声音。

目标：完成从检索到创作再入库。

### 练习 5：跑通云端协作

1. 启动云端开发服务。
2. 注册 Alice 和 Bob。
3. Alice 创建组织。
4. Alice 邀请 Bob。
5. Alice 上传素材到组织库。
6. Bob 登录后试听和检索。

目标：理解组织、角色和云端库。

### 练习 6：理解 C-C、B-B、B-C

1. C-C：创建个人库，把一个包同步到个人云库，再共享给另一个账号。
2. B-B：创建组织库，邀请另一个账号作为 editor，上传素材并查看编辑历史。
3. B-C：把同一个组织库授权给 viewer，让对方只负责检索、试听和下载。

目标：理解云协作不是单一上传，而是围绕“谁拥有、谁可用、谁可改、谁负责”建立资产关系。

## 24. 常见问题

### 为什么我的素材搜不到？

可能原因：

- 文件夹还没扫描完成
- 当前选择的是云端库而不是本地库
- 当前筛选了某个音频包
- 当前在收藏夹或最近使用视图
- 关键词不在已有元数据中
- 语义索引还没生成

建议先切回“全部资源”，清空筛选，再搜文件名的一部分。

### 为什么语义检索不可用？

可能原因：

- CLAP 依赖没安装
- 还没生成嵌入
- 模型加载失败
- 当前设备内存或 GPU 不足
- 索引任务正在运行或被暂停

先在设置页查看语义索引状态。

### 为什么波形不显示？

可能原因：

- ffmpeg/ffprobe 不可用
- 文件格式无法解码
- 文件已移动或删除
- 云端 mesh 资产持有节点离线

### 为什么拖到 DAW 失败？

本地素材：

- 检查源文件是否还在原路径。
- 检查 DAW 是否接受该格式。

云端素材：

- 检查是否有下载权限。
- 检查网络。
- 检查 token 是否过期。
- 检查云端文件是否存在。

### 为什么云端素材能看到但不能播放？

可能原因：

- 没有试听权限
- central 文件缺失
- mesh 持有节点离线
- 节点隧道未连接
- 云端服务地址配置错误

### 为什么合成台导出的声音没有某一层？

如果某个候选素材加载失败，当前版本可能不会明显提示。导出前建议先完整播放一遍合成，确认每条轨道都有声音。

### 为什么保存合成项目后出现多个项目？

当前保存更像“保存一个快照”。如果连续保存，可能产生多条记录。后续应通过项目 id 支持覆盖保存。

## 25. 给团队的推荐流程

### 小团队

1. 每个人先建立本地库。
2. 统一 UCS 和关键词习惯。
3. 配置 CLAP 语义索引。
4. 建一个组织云库放项目共享素材。
5. editor 负责上传和编辑，viewer 负责查找和下载。
6. 每周整理一次元数据和分组。

### 大团队

1. 使用 Postgres、Qdrant Server、对象存储和 Redis 部署云端。
2. 按项目或部门建立组织库。
3. 配置角色权限。
4. 规定上传命名、UCS、关键词和审计规范。
5. 使用管理后台观察用户、库、节点和审计状态。
6. 将本地个人库和组织库分开治理。

## 26. 最佳实践

- 不要一开始就导入所有历史素材。先用一两个高质量文件夹试跑。
- 先保证普通搜索、试听、拖拽都正常，再生成语义索引。
- 元数据字段宁可少而准，不要一次性乱填很多。
- 用虚拟分组整理包，不要轻易移动源文件。
- 重要文件写回元数据前，确认备份策略。
- 云端组织库适合共享源文件；个人分布式库适合个人设备间同步和持有登记。
- 合成台导出的成品建议重新补描述、关键词和 UCS 分类。
- 团队协作时明确谁是 admin、谁能 editor、谁只能 viewer。
- 做云端协作前，先判断关系类型：个人之间用 C-C，团队/组织项目用 B-B，组织开放给个人创作者用 B-C。

## 27. 一条完整的从零到产出路线

1. 启动 CoAudio。
2. 添加本地音效文件夹。
3. 等待扫描完成。
4. 在搜索框输入一个明确关键词。
5. 试听素材并收藏几条。
6. 打开设置，生成语义索引。
7. 输入一句自然语言找声音。
8. 使用查找相似扩展候选。
9. 编辑命中素材的描述和关键词。
10. 将素材拖入 DAW，确认工作流打通。
11. 打开合成台，用一句 brief 拆层。
12. 加入多条素材到时间线，调整音量和淡入淡出。
13. 导出新声音并重新入库。
14. 如果有团队需求，登录云端。
15. 创建组织库，上传项目素材。
16. 邀请成员，分配角色。
17. 用审计和历史跟踪协作修改。

完成这条路线后，你就已经掌握了 CoAudio 的核心价值：从混乱素材到可检索资产，从单条声音到组合设计，从个人硬盘到团队共享。
